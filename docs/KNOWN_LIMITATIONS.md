# Known Limitations

This project is currently focused on an MVP for a production practice report/demo.

## Current Limitations

- Demo data is limited.
- The primary seeded flow is Biology -> Genetics -> What is DNA?
- Authentication and full profile ownership are still simplified.
- The backend currently uses a shared demo user for lesson completion/profile behavior.
- Duels and Shop are planned modules, not full production systems.
- Quiz questions, battle logic, shop inventory, and purchases can be expanded later.
- Leaderboards, achievements, streaks, and some dashboard stats are currently presentation-oriented UI elements.
- The frontend includes polished fallback data so the demo flow still looks presentable if the backend is not running.

## Current Focus

The current priority is a stable, visually presentable MVP that demonstrates:

- A learning dashboard.
- A subject/topic/lesson map.
- A lesson detail flow.
- XP/coin rewards on completion.
- A player profile screen.
- Clear planned expansion paths for Duels and Shop.

## Future Improvements

- Add real authentication and per-user sessions.
- Add richer seed data and more subjects/topics/lessons.
- Add quiz interaction inside lessons.
- Implement Duels as quiz-based PvP battles.
- Implement Shop inventory, purchases, and cosmetic/effect ownership.
- Add automated tests for lesson completion and reward behavior.
